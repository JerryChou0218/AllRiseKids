# M24 — 產品架構與 UI/UX 重構

## 目標

把 AllRiseKids 從「所有功能集中在同一頁的兒童任務管理工具」，重構為具有明確主流程、孩子與家長雙模式、適合手機操作，並且更像遊戲的現實生活 RPG 成長系統。

## 範圍

- 孩子模式與家長模式明確分離。
- 孩子模式第一層導航收斂為：首頁、任務、冒險、獎勵、我的角色。
- 家長模式第一層導航整理為：總覽、審核、任務管理、獎勵管理、成長紀錄、設定。
- 首頁改為今日狀態與下一步，不再展示所有功能。
- 地圖/技能/裝備歸入冒險，商店/真實獎勵/背包歸入獎勵，狀態/成就/戰績歸入我的角色。
- 建立集中式功能解鎖設定與相容判斷。
- 初次使用流程清楚說明產品價值、角色建立、家長 PIN 與第一個任務。
- 保留既有資料格式、localStorage key、雲端同步、任務、EXP、金幣、審核、商店、背包、成就、交易、匯入/匯出。

## 驗收標準

- 孩子模式第一層導航只有五個入口。
- 家長功能不再作為孩子模式第一層入口。
- 家長模式有獨立六分頁 shell。
- 首頁只突出角色摘要、今日任務進度、下一個推薦任務、近期成長與解鎖預告。
- 任務頁有今日任務、挑戰任務、任務紀錄。
- 真實獎勵維持申請與家長確認流程。
- 功能解鎖條件集中於單一設定，且舊玩家已有資料不會被硬擋。
- 現有九套 Node 測試全部通過。
- 沒有新增資料 migration；既有 migration 與舊存檔仍可讀取。
