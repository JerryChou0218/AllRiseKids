# Milestone 14: 寬版 PC 佈局 + 頂部帳號/家長列 + 隱藏孩子台幣 + 擴充裝備獎勵

## Context
前一個里程碑：13。使用者回饋四點：
1. 1080p 要寬版 PC 佈局。
2. 切換帳號/登入登出、家長登入都改到視窗上方，不要從「家長」功能進入。
3. 孩子介面不應看到 NT 台幣價值（並詢問「本月已核准 NT$」依據）。
4. 多產生一些裝備道具與真實獎勵。

## Objective
把行動優先的窄欄佈局升級為桌面寬版（左側欄導覽），把帳號與家長入口移到固定頂部列，並讓台幣金額只在家長模式出現；擴充商店與獎勵內容。

## Scope
- **寬版佈局**：新增 `.app-content` / `.app-nav` CSS。桌面（`@media min-width:1024px`）將底部導覽改為左側欄（直式、圖示+文字），內容區 `max-w-5xl/6xl` 並左側留出側欄寬度；行動裝置維持底部導覽。grid 螢幕（商店/成就/任務）在寬螢幕顯示更多欄。
- **頂部列**：`<header class="topbar">` 固定於視窗頂端——左＝帳號（`avatarFor`＋名稱，點擊 `switchAccount()` 登出/切換，無帳號時隱藏）、中＝標題、右＝`🔒 家長`（`tryParent()`）。`renderTop()` 同步更新。
- **隱藏孩子台幣**：孩子狀態頁「金幣與獎勵概況」第三格由 `NT$ 月度估值` 改為「本月已兌換獎勵」次數。台幣（`monthlyApprovedTwd`）僅保留在家長模式「經濟摘要」。
- **擴充內容**：SHOP_ITEMS 新增 15 件（x1–x15，涵蓋各槽位與稀有度）共 58；DEFAULT_REWARDS 新增 10 個（r16–r25）共 25。

## 「本月已核准真實獎勵 NT$」依據（回答使用者提問）
= 本月（當月）中，家長在管理者模式按下「核准」的真實獎勵兌換申請，其 `estimatedTwd`（預估台幣價值）之總和。用來對照建議月預算 NT$900；超出僅提醒、不限制。此數字只在家長模式顯示。

## Acceptance Criteria
- [x] 桌面左側欄 + 寬版內容；行動底部導覽直式佈局。
- [x] 頂部列常駐帳號（登出/切換）與家長登入；切換不需進家長功能。
- [x] 孩子畫面無台幣；NT$ 僅家長可見。
- [x] 商店 58 項、真實獎勵 25 項。
- [x] exe 1080p 寬版啟動成功；smoke 75/75、desktop-save 4/4、accounts 10/10。

## Risk Notes
- 佈局以 CSS media query 切換，未改動畫面 JS 邏輯，測試不受影響（純視覺）。
- 寬版內容仍沿用現有元件；若日後要為個別頁面做 2 欄式資訊牆，可另立里程碑。
