# Milestone 8: 跨裝置雲端同步（提案，需家長/隱私審核）

## Context
前一個里程碑：7（PWA 安裝與離線快取）。
目前系統狀態：進度純存於本機 localStorage，README 明列限制「換瀏覽器/換電腦不會同步」。

## Objective
提供**可選**的雲端存檔同步，讓孩子能在不同裝置延續進度，同時保持未登入時 100% 本機、離線的現況。

## Scope
- 可選帳號/登入機制與雲端存檔（上傳/還原 `kidquest_data`）。
- 衝突處理（以最新或家長選擇為準）。
- 家長同意流程與資料最小化。

## Out of Scope
- 社群/排行榜等需公開資料的功能。
- 即時多人。

## Technical Approach
待規劃時決定後端（如 Serverless + 受管資料庫或 BaaS）。**此里程碑 `require_human_approval: true`**：先以 Mockup + 隱私評估通過後才實作。

## Acceptance Criteria
- [ ] 登入後可將存檔同步到雲端並於另一裝置還原。
- [ ] 未登入時遊戲行為與現在完全相同（純本機、可離線）。
- [ ] 兒童個資處理符合隱私原則：最小化蒐集、明確家長同意、可刪除。
- [ ] `node test/smoke.js` 維持 65/65 通過。

## Risk Notes
- 兒童個資（COPPA/GDPR-K 類）合規為最高風險，需人工審核與家長同意。
- 後端引入成本與維運；需評估是否真有必要或改用匯出/匯入存檔的輕量方案。
- 帳號安全與存檔被覆蓋的資料遺失風險。

## 實作決定（2026-06-13，待人工審核）
本里程碑採用 **Risk Notes 中的輕量方案：純本機「匯出/匯入存檔」**，而非架設帳號制雲端後端。理由：
- 此專案為純靜態、離線單檔，引入後端需基礎設施決策（後端選型、帳號系統、維運），屬高風險且需人工拍板。
- 匯出/匯入用本機 JSON 檔在裝置間轉移進度，**零資料蒐集、不上傳、無帳號**，比「最小化蒐集」更徹底，完全避開兒童個資合規風險。
- 對「換裝置延續進度」的真實使用情境（多為家長協助轉移）已足夠。

實作內容：管理者模式 → 設定新增「📤 匯出存檔 / 📥 匯入存檔」；匯出下載 `kidquest-save-YYYYMMDD.json`，匯入經家長確認後覆蓋本機存檔並重載（沿用既有 migrate 流程）。

**待人工決定**：若仍需「真正的帳號制雲端同步」（自動跨裝置、無需手動轉檔），那是一個獨立、需基礎設施與隱私評估的後續里程碑——核准本方案即視為以輕量方案結案 M8。
