# Milestone 8: 跨裝置雲端同步（提案，需家長/隱私審核）

## Context
前一個里程碑：7（PWA 安裝與離線快取）。
目前系統狀態：進度純存於本機 localStorage，README 明列限制「換瀏覽器/換電腦不會同步」。

## Objective
提供**可選**的雲端存檔同步，讓孩子能在不同裝置延續進度，同時保持未登入時 100% 本機、離線的現況。

## Scope
- 可選帳號/登入機制與雲端存檔（上傳/還原 `kidquest_data`）。
- 衝突處理（以最新或家長選擇為準）。
- 家長同意流程與資料最小化。

## Out of Scope
- 社群/排行榜等需公開資料的功能。
- 即時多人。

## Technical Approach
待規劃時決定後端（如 Serverless + 受管資料庫或 BaaS）。**此里程碑 `require_human_approval: true`**：先以 Mockup + 隱私評估通過後才實作。

## Acceptance Criteria
- [ ] 登入後可將存檔同步到雲端並於另一裝置還原。
- [ ] 未登入時遊戲行為與現在完全相同（純本機、可離線）。
- [ ] 兒童個資處理符合隱私原則：最小化蒐集、明確家長同意、可刪除。
- [ ] `node test/smoke.js` 維持 65/65 通過。

## Risk Notes
- 兒童個資（COPPA/GDPR-K 類）合規為最高風險，需人工審核與家長同意。
- 後端引入成本與維運；需評估是否真有必要或改用匯出/匯入存檔的輕量方案。
- 帳號安全與存檔被覆蓋的資料遺失風險。
